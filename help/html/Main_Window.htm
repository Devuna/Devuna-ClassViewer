Window Caption

 

The window caption displays the Clarion Version you are working with, the name of the program "Class Viewer", and the current Object Type that is being viewed in brackets.

 

 

Main Menu

 

The main menu provides a common access point for all the functionality in the program. It consists of five sub-menus.

 

File Menu

 

Clarion Versions
The list of available Clarion versions is created by examining the win.ini file. This is where you select the version of Clarion you want to work with. The first time you select a particular Clarion version, the software will scan the source files and create a database for that version of Clarion.
 
The databases are created in the folder from which the Class Viewer was launched. For this reason, it is a good idea to always run the Class Viewer program from the same folder (unless of course, you want to maintain several different databases). The location of the databases can be overridden by specifying the datafile path as the first command line parameter when launching the abcview.exe.
 
The databases are actually memory dumps of the queues used by the program. They are stored in a binary format (not TPS files) which is loaded and saved by the program as necessary. It is therefore not possible to provide direct access to the database which is why we have the "XML Export" menu option.
 
Exit
Choose this option to exit the program.

Expand All

View Menu

 

The first section of the View Menu allows you to select the type of object to be displayed on the Object List. The selection for the Object Type currently being viewed is disabled in the menu (Equates in the above screen shot). Your choices and their associated hot keys are as follows:

 
Classes
Select this option or press CtrL+L to view classes and their properties, methods, interfaces, and derived classes.
 
Interfaces
Select this option or press CtrL+I to view the interface declarations.
 
Call Tree
Select this option or press CtrL+T to view the method call tree. This view provides a summary of methods called.
 
 
 

The program does not have the ability to distinguish between overridden methods.

 
 
Structures
Select this option or press CtrL+U to view GROUP and QUEUE type declarations.
 
Equates
Select this option or press CtrL+E to view EQUATES and ITEMIZE declarations.
 
 

The second section of the View Menu allows you to perform common activities on the item currently highlighted in the Tree View.

 
Definition
Select this option or press CtrL+ENTER to view the definition of the item currently highlighted in the Tree View.
 
Help
Select this option or press CtrL+H to view the Clarion Help file documentation for the item currently highlighted in the Tree View.
 
Notes
Select this option or press CtrL+N to View or Add a Note to the item currently highlighted in the Tree View.
 
 
Tools Menu

 

Options...
Select this option or press CtrL+O to access the Edit Options dialog. This dialog allows you to configure many aspects of the program including default actions, additional files to be scanned, color preferences, and editor options. Please see the Edit Options help topic for a more detailed explanation.
 
Refresh Tree
Select this option or press CtrL+R or F5 to access the Scan Options dialog. This dialog allows you to configure the scanning process. You can choose to scan all files or use the program's Smart-Scan technology to scan only the files that have been modified since they were last scanned. Pressing ctrl+f5 forces the program to skip the Scan Options dialog and perform a Smart-Scan.
 
Add Class Wizard
Select this option or press CtrL+W to access the Add Class Wizard dialog. This dialog assists in creating derived classes based on user definable templates and options.
 
Browse Database
Select this option or press CtrL+B to access the Browse Database window. This window provides a structured view of the underlying data tables used by ClassViewer.
 
Statistics
Select this option or press CtrL+S to view summary statistical information regarding the data objects stored in the ClassViewer database. Press the OK button to close the window.
 
 
Favorites
Select this option to expand the list of favorites. This menu list is defined by the user in the Edit Favorites dialog. Use this list to provide easy access to your favorite programs, documents, web sites, etc. The favorites menu uses Windows associations to do its work, so anything that opens properly when you double-click on it in the Windows File Explorer should work on the menu.
 
 
Find Notes
Select this option or press CtrL+F to access the Find Notes dialog. This dialog displays a list of all notes in your database allowing you to easily find and review or update the existing notes in your database.
 
Clear History
Select this option to or press CtrL+Y to clear the Hyperlink History list. For more information about the Hyperlink feature, please see the Hyperlink Toolbar topic.
 
XML Export
Choose this option to display the XML Export menu.
 
 
Database to XML
Choose this option to export the data queues used by the program. The Export Database To XML dialog is displayed to allow the user to select which queues to export and the names of the files to be used.
 
Tree to XML
Choose this option to export the currently displayed tree to XML format. A file dialog is displayed to allow the user to select the XML filename.

Main Toolbar

 

The Main Toolbar provides buttons for most common activities. The button for the currently selected Object Type is latched and disabled. Click on a button above to find out more about its function.

 

 

Favorites Toolbar

 

The Favorites Toolbar is dynamically created from the items you have selected in the Favorites Menu list. The program uses icon [0] from the executable file, if specified. If the list entry is a document the program uses icon [0] from the program specified in the windows associations. The program will only place buttons on the toolbar if there is physical space between the Main Toolbar and the Hyperlink Toolbar.

 

 

Object Filter Toolbar

 

The Object Filter Tools allow you to filter the contents of the Object List depending on the type of Object you are currently viewing.

 

Category Filter

The Category Filter is available when viewing Class or Call Tree object types. The category is extracted from the !ABCIncludeFile(Category) comment line that identifies ABC Classes. For Non-ABC Classes the program also recognizes a !Category(Category) comment line. The category for a class can also be set via the Class Properties dialog.

 

 

Tree Filter Toolbar

 

The Tree Filter Tools allow you to filter the contents of the Tree View depending on the type of Object you are currently viewing.

 

Detail Level Filter

The detail level filter is available when viewing the Class object type and allows you to fine tune how much detail is presented in the tree view. Most classes are assigned a detail level of 1 when they are initially scanned. There are some basic classes [see list] that are assigned detail level 0. This was done to allow the program to present a simple view to the beginner while providing a feature for advanced users that permits fine tuning of the amount of detail presented in the tree view. This feature would be particularly useful in controlling the presentation of derived classes. The class detail level is maintained by the Class Properties dialog.

 

Protected Attribute Filter

The PROTECTED attribute filter is available when viewing Class and Call Tree object types. When checked, properties and methods of the class that have the PROTECTED attribute are included in the tree view and are colored according to the preferences defined by the Edit Options dialog. When unchecked, properties and methods of the class that have the PROTECTED attribute are excluded in the tree view.

 

Private Attribute Filter

The PRIVATE attribute filter is available when viewing Class, Call Tree, Structure, and Equate object types. When checked, properties, methods, structures and equates that have the PRIVATE attribute are included in the tree view and are colored according to the preferences defined by the Edit Options dialog. When unchecked, properties and methods of the class that have the PRIVATE attribute are excluded in the tree view.

 

Module Attribute Filter

The MODULE attribute filter is available when viewing the Class or Call Tree. When checked, properties and methods that are declared locally in the class definition module (normally the class.clw file) are included in the tree and are colored according to the preferences defined by the Edit Options dialog. When unchecked, properties and methods of the class that are defined inside the class definition module are excluded from the tree view.

 

Hyperlink Toolbar

 

 

The Hyperlink toolbar contains four buttons: the hyperlink button, back and forward navigation buttons and a Hyperlink History drop list to allow random navigation to previously hyperlinked references. A hyperlink is a quick way to navigate to a referenced object. Referenced objects are shown with an underline font in the tree view and can be easily hyperlinked to by double clicking on the reference in the tree view.

 

Hyperlink

The hyperlink button is enabled when a referenced object is selected in the tree view. Press the button to shift focus to the referenced object.

 

Back

The back button allows you to navigate backward through the hyperlink history list.

 

Forward

The forward button allows you to navigate forward through the hyperlink history list.

 

Hyperlink History

Click the hyperlink history button and a drop-down list of hyperlinked references is displayed. Select a reference from the list to bring it into focus.

 

 

Object List

The Object List is used to select the item of interest from the list of available choices. When viewing Class, Interface, or Call Tree object types, the list box header will contain the currently selected Class Filter.

 

The Class Filter is available when viewing Class, Interface, or Call Tree object types. When you click on the Object List Header, a dropdown list appears with the following choices:

 

All Classes

     Choose this selection to disable Class Type filtering.

ABC Classes

     Choose this selection to view only ABC Classes

Non-ABC Classes

     Choose this selection to view only Non-ABC Classes

 

Selecting an item in the Object List usually causes the focus of the Tree View to change to the selected item.

 

Right Click on the list to display Object List popup menu. There are two choices on this menu, the Add Class Wizard and the Wrapper Generator.

 

When viewing the Call Tree object type the format of the Object List changes to a tree with the Classes as root level nodes. Selecting a root level node will not cause a change in the Tree View. You must expand the Class in the list and select a class method before the Tree View changes.

 

The Object List is supported by an incremental locator feature. Just start typing the class name to quickly locate it in the list.

 

 

Locator Display

 

 

When active, the Locator icon and text are displayed directly below the Object List. To clear the locator just click on the Object List or press the up or down keys.

 

 

Tree View

 

Right Click on the Tree View to display the Popup Menu.

The Tree View provides a hierarchical view of the attributes of the item selected in the Object List.

 

The first column of the listbox will display the note icon if a note has been attached to that list item. Double Click to view.

 

The second column provides a tree view of properties, methods, interfaces, derived classes, structure components, and equates. The list items are color coded depending on their attributes and according to the color scheme defined by the user in the Edit Options dialog.

 

The mouse double click action is also defined by the user in the Edit Options dialog. The action can be configured to invoke the ascii viewer or the selected editor. If the user double clicks on a hyperlinked item (shown with underline font) then the focus of the tree view is shifted to the hyperlinked item.

 

The ToolTip for the selected item shows the module name and line number where the item is declared.

 

If a note icon is visible, holding the mouse over the icon will cause the first 255 characters of the note to be displayed as a tooltip.

 

Once you have scanned classes for earlier versions of Clarion, the program can identify classes that did not exist in the prior version. The 'new' classes are highlighted in bold font and have a slightly different icon.

Hyperlink

Source View

 

The Source View provides an ascii viewer for the item selected in the tree view. The program picks the file that the user would most likely want to view. As the user navigates the Tree View, the Source View changes to show the selected item in it's source context. The ascii viewer on the main window uses the scintilla open source editor control to provide color syntax viewing of the file. The styles used by this control can be set in the Edit Options dialog on the Viewer Tab. Viewer functions can be accessed from a popup menu. Right-click on the viewer display to get the menu to appear.

 

 

Legend

 

The legend appears below the Object List and Tree View panels. Pictured above are the various legend components. Not all legend components are visible at the same time; the legend is context sensitive. Colors for the Virtual, Protected, Private, and Module attributes can be set by the user in the Edit Options dialog.

 

 

Splitter Bar

The splitter bar is located between the Object List and the Tree View. When the mouse is over the splitter bar, the cursor changes to an EW Sizing Cursor . Press the left mouse button while the sizing cursor is active and the Splitter Bar will turn blue. Keep the left button pressed and move the mouse left or right ro resize the Object List and Tree View panels.

 

There is also a splitter bar between the Tree View and Source View panels to allow the user to resize the Tree View and Source View panels.

 

 

Exit Button

 

 

Press the exit button to exit the software.
